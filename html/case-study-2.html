Process Steps

The project comprised multiple Think-Make-Check iteration cycles, with different combinations of the below variables.

  • Think

    • Problem Statement
    • Business Outcomes
    • Personas
    • User Outcomes
    • Hypothesis Statements
    • User Interviews
    • Task Flow Analysis
    • Content Audit
    • Retrospectives
    • Data Hierarchy (IA)
    • Stakeholder Interviews

  • Make

    • Concierge MVP
    • A/B Testing
    • User Testing
    • Sitemap
    • User Flows
    • Sketches
    • Wireframes
    • Responsive Layouts
    • Refactoring

  • Check

    • Qualitative User Feedback
    • Engagement Metrics
    • User Testing Results
    • Business Outcome Metrics

Competitive Analysis

Finding my way in the marketplace

I first compiled an exhaustive review of existing mobile apps. I found that each had their own strengths, but none of them had the key combination that I was fundamental to Bróg’s unique offering, namely (1) a sole focus on pronunciation, (2) real feedback from other users, (3) a daily task-based usage model.
